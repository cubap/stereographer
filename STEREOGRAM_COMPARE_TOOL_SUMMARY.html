IIIF Stereogram Compare Tool - Springboard Summary

This document describes the local scaffold created for a standalone tool repo. It is intentionally practical and implementation-focused so you can resume quickly in a new context.

Created Local Files

  • stereogram-compare-tool/index.html - app shell, form inputs, preview regions, alignment controls, export actions.
  • stereogram-compare-tool/styles.css - base layout and visual styling.
  • stereogram-compare-tool/app.js - initialization flow, IIIF parsing, preview update hooks, export stubs, RERUM POST wiring.

Implemented Baseline

  • Accepts input from URL text or pasted JSON.
  • Attempts to resolve resource type: Image, Canvas, Manifest.
  • Supports ?iiif-content= parameter and base64url decode fallback.
  • Overlay controls: blend, nudge X, nudge Y, blink start/stop.
  • Export actions wired: download manifest, copy base64url iiif-content, save Canvas or Manifest to RERUM create endpoint.
  • Selector mode toggle prepared (ImageApiSelector vs direct image region URL).

Gaps To Finish Next

  • Add draggable and resizable crop boxes over source preview, then keep region state in sync.
  • Apply left and right crop regions to overlay previews so they render as true cropped views, not full image placeholders.
  • Generate two painting annotations (left and right) in exported Canvas/Manifest, each with timeline targets.
  • Robustly support input initialization from Content State payload object, not just URL extraction.
  • Improve CORS and fetch error handling for remote JSON and images.
  • Add auth flow for RERUM create if token required by runtime environment.

Suggested Next Architecture Pass

State Modules

  • resourceState: loaded IIIF resource, resolved source image/service, dimensions.
  • regionState: left and right region objects with x/y/w/h, plus drag interaction state.
  • viewState: blend, nudges, blink speed, selector mode.
  • exportState: generated canvas/manifest payload and encoded iiif-content strings.

Service Modules

  • iiifResolver: resource detection and normalization.
  • manifestBuilder: direct image body mode and ImageApiSelector mode.
  • contentStateCodec: encode/decode base64url content state.
  • rerumClient: create/update wrappers with error normalization.

Export Mode Requirements

  • Direct image mode: body is type: Image with region URL in id.
  • ImageApiSelector mode: body is SpecificResource with source and selector { type: "ImageApiSelector", region: "x,y,w,h" }.
  • Offer both Canvas-only and Manifest exports.
  • Offer base64url for iiif-content query parameter usage.

Quick Local Run

Open stereogram-compare-tool/index.html directly in a browser
or serve the folder with a static server for cleaner fetch behavior.

Context Handoff Notes

When you switch to a dedicated repo context, copy the folder as-is, then split app.js into modules and add tests around resolver and export logic first.
